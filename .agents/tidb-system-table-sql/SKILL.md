# TiDB System Table SQL

## Workflow
1. Collect target context first: `table_schema`, `table_name`, whether table is partitioned, whether TiFlash is involved.
2. Open `references/sql-cookbook.md` and jump to the matching section.
3. Replace placeholders (`<database_name>`, `<table_name>`, `<partition_name_pattern>`, `<table_id>`, `<job_id>`, `<task_id>`) before execution.
4. Prefer read-only SQL first, then execute mutating SQL only after explicit user confirmation.

## Quick Navigation
- Region peers and leader distribution: `Region Peers Distribution`
- Store type filtering (tikv/tiflash/tiflash_compute): `Store Filtering`
- Table ID and index ID lookup: `Table and Index ID Lookup`
- TiFlash replica query / generate ALTER SQL: `TiFlash Replica`
- TiFlash / TiKV storage usage estimation: `Storage Usage`
- DDL queue and cancel command: `DDL Jobs`
- DXF and TiCI progress: `Task Progress`

## Safety Rules
- Default to read-only SQL statements.
- Treat the following as mutating operations and require explicit confirmation before execution:
  - `ADMIN CANCEL DDL JOBS ...`
  - `ALTER TABLE ... SET TIFLASH REPLICA ...` (generated by query output)

## Notes
- `TiKV disk storage space estimation` SQL requires replacing `<database_name>` and `<table_name>`.
- The cookbook preserves SQL and examples for quick reuse.
